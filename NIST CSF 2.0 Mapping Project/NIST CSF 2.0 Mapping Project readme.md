# NIST CSF 2.0 Mapping Project for Oil & Gas Portfolio

## Purpose
Map the portfolio deliverables to the NIST Cybersecurity Framework 2.0 functions: Govern, Identify, Protect, Detect, Respond, and Recover.

## Portfolio Mapping
| CSF 2.0 Function | Portfolio Artifact | Oil & Gas Example Outcome |
|---|---|---|
| Govern | AI Governance Policy Framework | AI and cyber risk expectations are established and approved |
| Identify | Risk Assessment Template | Critical IT, OT, cloud, and vendor assets are identified and assessed |
| Protect | Security Awareness Program | Personnel understand phishing, remote access, AI, and data protection rules |
| Detect | Incident Response Plan | Monitoring and reporting channels support early detection |
| Respond | Incident Response Plan | Cyber incidents are escalated, contained, and communicated |
| Recover | Incident Response Plan | Recovery actions restore operations and capture lessons learned |
| Govern / Supply Chain | Vendor Risk Assessment | Third-party risk is assessed before onboarding and periodically reviewed |

## How This Project Shows Value
This mapping demonstrates that the portfolio is not just a set of templates. It shows how governance, risk assessment, training, incident response, vendor risk, and framework alignment work together as a practical cybersecurity management system.
